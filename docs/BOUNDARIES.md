# Design-anchor Directory Boundaries (Required Reading)

In repositories using Design-anchor, strictly follow this separation to prevent mixing app code with the component library and to prevent AI from writing pages into wrong directories.

## 1. Application Code

- **Location**: Your project's own app directory, typically `src/` (or `app/`, `packages/web/src`, etc.).
- **Content**: Routes, pages, features, API calls, business state, layout composition.
- **Forbidden**: Do NOT write production business pages inside `.anchor/`.

## 2. Component Library & Design Assets (`.anchor/`)

- **Location**: Hidden directory **`.anchor/`** at project root (generated by `anchor init` / `anchor start`).
- **Content**: Base/Business component source, `.demo.tsx` previews, Design Tokens, `src/anchor/schema/*.spec.json`, Portal config, sync scripts.
- **Purpose**: Local design workbench + versionable component library sub-project — NOT the app root.

## 3. AI Governance (`.cursor/` and other tool configs)

- **Location**: Project root **`.cursor/`** (rules, mcp, optional hooks), plus `CLAUDE.md`, `.windsurfrules`, `.github/copilot-instructions.md`.
- **Content**: `anchor.mdc` rules, MCP pointing to `.anchor` paths.
- **Note**: Separated from app source; commit strategy is up to your team (usually committed alongside `.anchor` or partially gitignored for generated files).

## 4. Import Conventions (Recommended)

- App code referencing UI: **Prefer** the `@design` path alias pointing to `.anchor`'s barrel (see `ANCHOR_INTEGRATION.md`), or equivalent relative paths.
- App code **must NOT** deep-import unexported implementation files from inside `.anchor`, unless the team explicitly allows it.

## 5. Monorepo Structure

For monorepo projects (pnpm workspace / npm workspaces), recommended structure:

| Area | Path | Responsibility |
|------|------|---------------|
| **UI source of truth** | `packages/business-ui/` | Components, variants, Tokens, manifest |
| **Design-anchor integration** | `.anchor/` (or `apps/anchor/`) | Portal, Vite dev server, sync scripts |
| **Business apps** | `apps/web/` etc. | `workspace:*` depends on `@acme/business-ui` |

Key constraints:
- `packages/business-ui` **must NOT** depend on `.anchor` (one-way: anchor/apps → business-ui).
- All apps `import { Button } from '@acme/business-ui'` — never copy component implementations into `apps/web/src/`.
- Upstream npm kit synced to `packages/business-ui` via `anchor upgrade`.

## 6. Quick Reference

| Directory | Who edits | What goes there |
|-----------|-----------|-----------------|
| Project `src/` (etc.) | Feature development | Pages and business logic |
| `.anchor/` | Design system / component maintenance | Components, Tokens, Specs, Portal |
| `.cursor/` + tool configs | Engineering / AI config | Rules and MCP |
| `packages/business-ui/` | Component source (monorepo) | Components + Tokens + manifest |
